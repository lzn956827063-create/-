项目概述

CodePilot 的核心设计理念是 AI Agent 自主决策。传统 AI 问答是"固定上下文 + 大模型回答"，而 CodePilot 让 AI 像人类开发者一样边探索边回答：AI 先调用 get_project_overview 了解项目结构，再调用 search_code 搜索关键词，找到目标文件后用 read_file 深入阅读代码，在充分理解代码后才能给出精准答案。

项目共实现了 3 个专业 Agent，根据任务类型自动切换探索策略：

💬 智能问答 Agent（ReAct 循环）

开放式答疑，用户可以问任意问题（"登录在哪？""用的是哪个 ORM？"）。Agent 最多执行 8 轮工具调用，前端实时展示每一步的思考过程。回答结构为三段式：📌 结论 + 🔍 关键发现 + 💡 补充建议。

🔎 项目分析 Agent（探索 → 生成两阶段）

先自由探索源码（入口文件、路由、数据模型、核心逻辑、配置文件），收集关键上下文；再调用 DeepSeek 生成固定 6 维度 JSON 报告：项目介绍、主要功能、技术栈、系统架构、核心模块、开发建议。结果持久化到数据库。

📖 README 生成 Agent（探索 → 生成两阶段）

专注文档化：探索安装命令、运行方式、功能说明、配置方法等"可复现"的信息，基于实际代码生成专业 README.md，杜绝编造内容。

项目架构

Vue 3 前端 ←→ FastAPI 后端 ←→ DeepSeek API
API 层 → Service 层 → Agent 层 → Tool Executor
Agent 循环：思考 → 调用工具 → 观察结果 → 再思考 → 给出答案
5 个工具：search_code / read_file / list_directory / read_config / get_project_overview

Agent 工作流程

  1. 用户提问：比如"登录功能在哪里实现的？"
  2. Agent 思考：需要先了解项目结构
  3. 调用工具：get_project_overview() → 获取目录树和技术栈
  4. 观察结果：发现项目有 auth 目录
  5. 继续探索：search_code("login") → 搜索相关文件
  6. 深入阅读：read_file("app/auth/routes.py") → 确认具体实现
  7. 给出答案：信息足够，生成结构化回答（文件路径 + 关键代码 + 说明）

Agent 工具集

工具名称功能关键参数
get_project_overview获取项目全局信息（目录树、技术栈、统计）无参数
search_code按关键词搜索代码，返回匹配文件+行数+上下文片段keyword, file_pattern
read_file读取指定文件，支持起止行号，路径穿越防护file_path, start_line, end_line
list_directory列出目录内容（文件和子目录）path
read_config读取项目配置文件摘要（依赖清单、技术栈）无参数

技术亮点

  • ReAct Agent 架构：Thought → Action → Observation 循环，AI 根据每次观察结果动态决策下一步，非写死工作流
  • Function Calling 零依赖：直接使用 DeepSeek 原生 Function Calling API（OpenAI 兼容格式），不依赖 LangChain 等重框架
  • 模块化设计：api / service / parser / ai / agent 五层分离，Agent 层通过抽象的工具接口与业务层解耦
  • 异步全链路：FastAPI + httpx + aiosqlite 全异步，Agent 循环中多次 API 调用不阻塞其他请求
  • 安全机制：路径穿越防护（所有文件操作限制在项目目录内）、文件大小/类型过滤、DeepSeek 响应 Unicode 清理
  • 探索-生成两阶段：分析 Agent 和 README Agent 采用统一的 Phase 1（自由探索）+ Phase 2（结构化输出）模式，可复用为基类
